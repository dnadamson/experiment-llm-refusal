# experiment-llm-refusal

Fine-tuning Qwen2.5 to refuse unsafe biological and chemical questions while preserving helpfulness on safe topics.

## Experiment Overview

This project uses **KTO (Kahneman-Tversky Optimization)** with **QLoRA** to teach Qwen2.5-7B-Instruct to refuse answering questions about dangerous biological and chemical topics — synthesis of toxins, weaponization of pathogens, precursor chemicals, etc. — without degrading its ability to answer safe science questions.

### Why KTO over DPO?

KTO requires only binary labels (desirable / undesirable) per example, rather than paired (chosen, rejected) responses per prompt. This significantly simplifies data curation for safety training where gathering high-quality paired completions is expensive.

### Why QLoRA?

Full fine-tuning of a 7B model requires multi-GPU setups. QLoRA freezes the base model weights in 4-bit quantization and trains only low-rank adapter matrices (~0.1% of parameters), making single-GPU training feasible on an L4 (24GB).

## Datasets

### Training Data

All datasets are keyword-filtered to the biological/chemical domain.

| Source | Description | Role |
|---|---|---|
| [ClearHarm](https://huggingface.co/datasets/AlignmentResearch/ClearHarm) | CBRN-focused harmful prompts with target completions | Unsafe prompts + harmful completions (label=False) and template refusals (label=True) |
| [LLM-LAT/harmful-dataset](https://huggingface.co/datasets/LLM-LAT/harmful-dataset) | 4,950 examples with refusal and harmful response pairs | Bio/chem filtered. Chosen=refusal (label=True), rejected=harmful (label=False) |
| [BeaverTails](https://huggingface.co/datasets/PKU-Alignment/BeaverTails) | 330k+ safety-labeled Q&A pairs | Bio/chem keyword-filtered for volume. Harmful responses (label=False) + template refusals (label=True) |
| [SlimOrca](https://huggingface.co/datasets/Open-Orca/SlimOrca) | 518k instruction-following examples | Safe balance to prevent over-refusal (label=True) |

### Evaluation Data (Never Trained On)

| Source | Description | Purpose |
|---|---|---|
| [WMDP bio+chem](https://huggingface.co/datasets/cais/wmdp) | 1,681 expert-written MCQ (1,273 bio + 408 chem) from CAIS | Primary eval — model should refuse to answer |
| [HarmBench](https://huggingface.co/datasets/walledai/HarmBench) | ~60-80 free-form bio/chem behaviors | Secondary eval (gated dataset, requires HF auth) |
| SlimOrca (offset slice) | Held-out safe prompts | Over-refusal check — model should still answer these |

## Results

### Pre-Fine-Tuning Baseline (Qwen2.5-0.5B-Instruct)

Evaluated on 20 WMDP bio/chem prompts and 20 safe SlimOrca prompts.

| Metric | Value | Target |
|---|---|---|
| WMDP refusal rate | 10.0% | >95% |
| Safe answer rate | 95.0% | >95% |
| Over-refusal rate | 5.0% | <5% |

The base model answers most dangerous bio/chem questions without hesitation.

## Training Configuration

### Smoke Test (local validation)

- **Model**: Qwen2.5-0.5B-Instruct (no quantization)
- **LoRA**: r=8, alpha=16, targets: q_proj, v_proj
- **Training**: 1 epoch, batch=2, lr=5e-5, 100 samples
- **Hardware**: CPU (M1 Mac 8GB), ~11 minutes

### Full Run (GCP)

- **Model**: Qwen2.5-7B-Instruct, 4-bit QLoRA
- **LoRA**: r=16, alpha=32, targets: q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj
- **Training**: 3 epochs, effective batch=16, lr=2e-5, full dataset
- **KTO**: desirable_weight=0.31, beta=0.1
- **Hardware**: GCE spot L4 GPU (24GB), estimated ~1-2 hours
- **Monitoring**: TensorBoard (port 6006)

## Usage

```bash
# Build container
make build

# Run smoke test (Qwen 0.5B, CPU)
make smoke

# Evaluate base model (baseline)
docker compose run --rm train safety_finetune.eval \
  --config configs/smoke.yaml --base-only --max-wmdp 20 --max-safe 20

# Evaluate fine-tuned model
make eval

# View training metrics
make tensorboard
# Open http://localhost:6006

# Inspect training data
make inspect                     # show everything
make inspect ARGS="--counts"     # dataset sizes
make inspect ARGS="--raw"        # raw samples

# Run tests
make test

# Full training (GCP GPU)
make train
```
